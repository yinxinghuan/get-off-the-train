# 《挤下地铁》视觉 QA

## Context

- 构建：本地 Vite production build / 2026-07-19。
- 目标：新游戏首轮与修复后复验；对照 `doc/requirements.md` 和 `doc/visual.md`。
- 视口：390×844 主目标，320×568 窄屏，英文长文案。
- 证据：`_qa/ui/first-pass/`、`_qa/ui/recheck/`。

## Executive assessment

- 2026-07-19 镜头重做最终决定：Pass；用户指出的“腐蚀/俯视感、主角找不到、出口方向反直觉、固定摇杆和开始菜单”均已关闭。
- 最强项：人群、扶手、座椅和广告共同形成可识别的无顶棚早高峰车厢，漫画 HUD 与昏暗 3D 场景分层清楚，主角局部光成为稳定空间焦点。
- 最大风险：首轮相机把长车厢横向裁切，远端门区不可见，属于 P1 目标识别问题。
- 原版首轮：P0 0 / P1 1 / P2 2；镜头重做首轮：P0 0 / P1 4 / P2 1；最终：P0 0 / P1 0 / P2 0。

## Findings and iteration

### 明亮车厢、无人物描边与右侧下车门复验

1. **P1 / 场景过暗**：上一版 `straight-route/level-4-lighting-320x568.png` 中后排乘客、座椅和侧墙大面积落入死黑。最终把 ACES 曝光从 0.82 提至 1.05、半球光从 0.48 提至 0.78、暖主光从 1.15 提至 1.72，并提高 5 组车厢灯的基准强度和半径；相同第 4 关证据为 `straight-route/level-4-bright-side-door-320x568.png`。
2. **P2 / 主角身体描边过重**：关闭主角 `makeCharacter()` 的外描边分支，头、身体、四肢和背包不再包黑色放大网格；保留随模型移动的双层脚底光环、暖色 PointLight、红背包与“你”标签。
3. **P1 / 车门位置不像地铁**：下车门从正前方端墙移到末段右侧壁，端墙恢复封闭。右上转向出口提示与地面折线箭头先引导向上，再引导向右穿门。
4. **P0 / 侧门可完成性**：关闭开发服务器热更新状态并完整重启后，基于玩家实时位置的折线路线在首关真实触发“挤出来了”，剩余 11.2 秒、0 次摔倒；证据 `straight-route/level-1-side-door-clear-320x568.png`。侧门引导区加入小幅碰撞半径容错，避免人群把玩家中心挤在门框边缘。
5. **镜头微调 / 增加俯角**：相机高度从 4.8 提至 5.35，视点从 1.0 降至 0.85，只改变纵向俯角，位置与视点的 `z` 仍相同。第 4 关能看到更多地面、人群缝隙和主角随身光环，证据 `straight-route/level-4-bright-side-door-320x568.png`；首关右侧红色门框、封闭端墙和转向路径证据为 `straight-route/level-1-bright-side-door-320x568.png`。

### 全人物呼吸弹跳复验

1. **P2 / 人物站立僵硬**：对照本地《Block Hop》的 idle bounce，把 `abs(sin(t*3.2+phase))` 引入主角、普通乘客和怪物；主角振幅 0.075、其他人物 0.045 world-unit，手臂随呼吸抬起、双腿轻微换重心。
2. **连续帧证据**：`straight-route/level-4-breath-bounce-a-320x568.png` 与 `straight-route/level-4-breath-bounce-b-320x568.png` 间隔 240 ms。主角身体、背包、脚底光环和局部光整体随人物上下移动；周围乘客使用独立 `phase`，没有整车同步弹跳。
3. **状态优先级**：移动速度叠加最高 0.065 world-unit 的 7 Hz 步伐弹跳；摔倒立即关闭呼吸与步伐高度，避免倒地角色继续上下浮动。减弱动态模式把全部环境弹跳降至 35%。

### 完全正向路线与可完成性复验

1. **P1 / 仍有侧向透视**：上一版相机沿出生点到侧门的对角线观察，虽然主角在下方，车厢仍向右偏斜。最终改为相机位置与视点共享同一 `z`，水平视线严格为世界 `+X`；证据 `straight-route/idle-player-exit-320x568.png`。
2. **P1 / 主角仍可能丢失**：只靠酸黄外套与红背包不足以对抗 39 人遮挡。最终叠加固定“你 / YOU”漫画标签、浅色头部和加粗酸黄脚底环；第 4 关仍可一眼定位，证据 `straight-route/level-4-player-320x568.png`。
3. **P0 / 旧侧门无法稳定结算**：旧逻辑先把玩家 `x` 限制在车厢内，再要求从侧面越过 `z` 边界，拥挤和晃动后容易卡在门口。最终把门移到正前方端墙，只有门宽内玩家可越过 `x` 边界，`x>8.63` 立即结算。
4. **功能复验 / 首关**：开发自动前进输入在 7.7 秒内真实触发“挤出来了”，剩余 12.3 秒、0 次摔倒；证据 `straight-route/level-1-clear-320x568.png`。
5. **功能复验 / 第二关**：同一持续前进输入在 24 人与周期晃动下仍真实触发结算，剩余 6.0 秒、累计 5,442 分；证据 `straight-route/level-2-clear-320x568.png`。这排除了只在空旷首关偶然通过的情况。

### 界面漫画 / 场景照明分层复验

1. **方向修正**：美式漫画只应用于 HUD、提示、摇杆和结算；3D 地铁保持原有低多边形场景，不叠漫画滤镜。
2. **拥挤关证据**：`straight-route/level-4-lighting-320x568.png` 为 320×568、第 4 关 39 名乘客实机截图。车厢整体曝光压低，HUD 仍保持奶油纸张、粗墨边和硬阴影；两种视觉语言没有混在同一材质层。
3. **主角空间识别**：主角同时由暖色局部照明、红色背包、双层脚底光环与“你”标签识别。拥挤状态下主角身体和脚下地面明显比周边亮，仍为画面第一焦点。
4. **摇晃灯光**：5 组车厢灯附着于车体并参与周期滚转；灯强以慢速呼吸变化，冲击峰值只让部分灯短暂变暗，不使用全屏闪白或高频闪烁。

### 第三人称即时开玩重做

1. **P1 / 镜头语义**：原正交俯视镜头像看模型沙盘，不像玩家在人群里向前挤。修复为 55° 透视、主角背后上方的跑酷式跟随相机，沿“出生点→车门”路线指数平滑；证据 `camera-rework/follow-playing-320x568.png`。
2. **P1 / 主角识别**：原奶油衬衫与普通乘客近似，拥挤关会丢失。修复为唯一酸黄外套＋朱红背包＋粗黑描边＋脚底墨环，且相机把主角锁在下方中央；第 4 关 39 人仍第一眼可见，证据 `camera-rework/crowded-level-4-320x568.png`。
3. **P1 / 出口方向**：原路线读成向屏幕下方或侧面移动。修复后屏幕上方恒为路线前方，并在 HUD 下设置酸黄 SVG 上箭头“出口”；拖动映射中屏幕向上直接投影到车门前向量。
4. **P1 / 开场与控制**：删除封面开始菜单和左下固定摇杆。页面加载即显示真实车厢，首次输入前计时冻结；任意位置落指生成 108 px 临时摇杆，松手隐藏。
5. **P2 / 引导**：删除常驻教程文字卡，改为 4.5 秒循环的墨线幽灵手指，从下向上演示拖动；首次真实触摸立即消失，证据 `camera-rework/instant-guide-320x568.png`。
6. **高反馈复验**：第三人称镜头下的周期预警仍能同时保留主角、出口和人群上下文，朱红提示不依赖颜色单独传意；证据 `camera-rework/sway-warning-320x568.png`。

### 原版历史修复

1. **P1 / 首关场景**：`first-pass/game-390x844.png` 只看到车厢一段，出口不在画面内。影响是玩家无法在第一眼建立目标与路径。修复为正交相机从 `(14,15,0.4)` 沿长轴观察，证据 `recheck/game-camera-390x844.png`。
2. **P2 / 输入映射**：相机旋转后，摇杆世界轴与屏幕轴不一致。修复为屏幕向下映射车厢 `+X`、屏幕向右映射 `-Z`。
3. **P2 / 主角识别**：高密度人群中奶油衬衫仍可能被遮挡。修复为主角脚底添加墨黑外环与酸黄内环，倒地时随角色一起旋转但不改变碰撞。
4. **P1 / 开场窄屏**：首版重复显示完整海报标题和超大本地标题，320×568 时内容拥挤。修复为完整方形叙事海报占上半屏，本地标题、无限模式、CTA 与榜单压入清楚的下半屏；证据 `recheck/start-endless-320x568.png`。
5. **P1 / 排行榜站外态**：必须避免假榜单与无法操作的玩家行。最终站外只显示 AlterU 入口，平台内才读取真实头像、姓名、最高分；证据 `recheck/leaderboard-320x568.png`。
6. **P2 / 英文长文案**：`ENDLESS CARS`、`LEADERBOARD` 与 `REDUCE MOTION` 在 320×568 均完整；证据 `recheck/start-en-320x568.png`。
7. **后期人群**：开发环境直达第 4/6 关复验了 39–48 人、循环车厢、怪兽 roster 与无限 HUD；证据 `recheck/monster-level-4-320x568.png`、`recheck/monster-level-6-320x568.png`。

## Foundation audit

- 功能 Emoji：0。
- 图标：统一 24×24、2.1 px 墨线 SVG。
- 触控目标：整个场景为动态摇杆输入区，摇杆可视环 108 px；暂停 48 px；结果主按钮 50–56 px。
- 对比：旧纸/墨黑、酸黄/墨黑、朱红/旧纸均配合边框或文字，不以颜色单独传意。
- 状态：场景内待操作、进行、预警、站稳、暂停、过关与失败均有明确恢复操作；无限模式没有伪造的“全通关”；音频和榜单网络失败均非阻断。
- 减弱动态：系统偏好自动开启，可在暂停面板切换；幽灵手指在减弱动态下保持静态姿势和文字说明。

## Final verification

- 320×568 的即时引导、第三人称首关与第 4 关高密度人群已按相同尺寸复验；旧版俯视证据保留用于前后对照。
- 正式海报已分别检查 1024×1024 与 160×160，标题位于顶部安全区，主体冲向黄色车门的冲突在缩略图仍可读。
- 严格 UI foundation 审计通过：功能 Emoji 0，功能图标均为同套 SVG。
- 最终平均分 4.1，所有类别不低于 4；可发布。
