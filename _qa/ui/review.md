## Findings and iteration

### 第三人称即时开玩重做

1. **P1 / 镜头语义**：原正交俯视镜头像看模型沙盘，不像玩家在人群里向前挤。修复为 55° 透视、主角背后上方的跑酷式跟随相机，沿“出生点→车门”路线指数平滑；证据 `camera-rework/follow-playing-320x568.png`。
2. **P1 / 主角识别**：原奶油衬衫与普通乘客近似，拥挤关会丢失。修复为唯一酸黄外套＋朱红背包＋粗黑描边＋脚底墨环，且相机把主角锁在下方中央；第 4 关 39 人仍第一眼可见，证据 `camera-rework/crowded-level-4-320x568.png`。
3. **P1 / 出口方向**：原路线读成向屏幕下方或侧面移动。修复后屏幕上方恒为路线前方，并在 HUD 下设置酸黄 SVG 上箭头“出口”；拖动映射中屏幕向上直接投影到车门前向量。
4. **P1 / 开场与控制**：删除封面开始菜单和左下固定摇杆。页面加载即显示真实车厢，首次输入前计时冻结；任意位置落指生成 108 px 临时摇杆，松手隐藏。
5. **P2 / 引导**：删除常驻教程文字卡，改为 4.5 秒循环的墨线幽灵手指，从下向上演示拖动；首次真实触摸立即消失，证据 `camera-rework/instant-guide-320x568.png`。
6. **高反馈复验**：第三人称镜头下的周期预警仍能同时保留主角、出口和人群上下文，朱红提示不依赖颜色单独传意；证据 `camera-rework/sway-warning-320x568.png`。

### 原版历史修复

1. **P1 / 首关场景**：`first-pass/game-390x844.png` 只看到车厢一段，出口不在画面内。影响是玩家无法在第一眼建立目标与路径。修复为正交相机从 `(14,15,0.4)` 沿长轴观察，证据 `recheck/game-camera-390x844.png`。
2. **P2 / 输入映射**：相机旋转后，摇杆世界轴与屏幕轴不一致。修复为屏幕向下映射车厢 `+X`、屏幕向右映射 `-Z`。
3. **P2 / 主角识别**：高密度人群中奶油衬衫仍可能被遮挡。修复为主角脚底添加墨黑外环与酸黄内环，倒地时随角色一起旋转但不改变碰撞。
4. **P1 / 开场窄屏**：首版重复显示完整海报标题和超大本地标题，320×568 时内容拥挤。修复为完整方形叙事海报占上半屏，本地标题、无限模式、CTA 与榜单压入清楚的下半屏；证据 `recheck/start-endless-320x568.png`。
5. **P1 / 排行榜站外态**：必须避免假榜单与无法操作的玩家行。最终站外只显示 AlterU 入口，平台内才读取真实头像、姓名、最高分；证据 `recheck/leaderboard-320x568.png`。
6. **P2 / 英文长文案**：`ENDLESS CARS`、`LEADERBOARD` 与 `REDUCE MOTION` 在 320×568 均完整；证据 `recheck/start-en-320x568.png`。
7. **后期人群**：开发环境直达第 4/6 关复验了 39–48 人、循环车厢、怪兽 roster 与无限 HUD；证据 `recheck/monster-level-4-320x568.png`、`recheck/monster-level-6-320x568.png`。

## Foundation audit

- 功能 Emoji：0。
- 图标：统一 24×24、2.1 px 墨线 SVG。
- 触控目标：整个场景为动态摇杆输入区，摇杆可视环 108 px；暂停 48 px；结果主按钮 50–56 px。
- 对比：旧纸/墨黑、酸黄/墨黑、朱红/旧纸均配合边框或文字，不以颜色单独传意。
- 状态：场景内待操作、进行、预警、站稳、暂停、过关与失败均有明确恢复操作；无限模式没有伪造的“全通关”；音频和榜单网络失败均非阻断。
- 减弱动态：系统偏好自动开启，可在暂停面板切换；幽灵手指在减弱动态下保持静态姿势和文字说明。

## Final verification

- 320×568 的即时引导、第三人称首关与第 4 关高密度人群已按相同尺寸复验；旧版俯视证据保留用于前后对照。
- 正式海报已分别检查 1024×1024 与 160×160，标题位于顶部安全区，主体冲向黄色车门的冲突在缩略图仍可读。
- 严格 UI foundation 审计通过：功能 Emoji 0，功能图标均为同套 SVG。
- 最终平均分 4.1，所有类别不低于 4；可发布。
