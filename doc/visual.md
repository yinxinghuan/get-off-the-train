# 《挤下地铁》视觉圣经

> 已锁定方向：界面延续便利商店 / 《午夜验客》的美式漫画语言；人物保留现有低多边形 roster，3D 地铁车厢采用写实 PBR 材质与轨道交通结构，用昏暗摇晃的车厢灯、出口引导光和主角局部光建立空间。漫画属于界面，不是覆盖场景的滤镜。

## 1. Visual thesis

- 游戏与受众：移动端短局动作玩家，熟悉早高峰但希望获得夸张喜剧释放。
- 情绪承诺：昏暗末班车里的真实压迫感，被清晰有力的漫画 HUD 切成紧张但带喜剧感的节拍。
- 一句话视觉命题：一节被摇晃顶灯切出明暗层次的无顶棚地铁里，主角带着一圈暖光挤向门口，漫画票据只负责告诉玩家时间与危险。
- 标志性视觉时刻：车身先倾、吊环齐摆，朱红“哐——！”格砸进画面，全车乘客像多米诺骨牌般互撞摔倒。
- 三个必须品质：车厢细节可信、动作一眼可读、混乱仍有视觉层级。
- 三个避免方向：默认 Three.js 灰盒、日式二次元、霓虹赛博地铁。

## 2. Composition and camera

- 竖屏 320×568–430×932；3D Canvas 全屏，DOM HUD 自适配，不整页缩放。
- 55° 透视相机位于主角后方 5.2 world-unit、高 6.05 world-unit，严格沿车厢 `+X` 中轴看向前方 4.8 world-unit、视点高度 0.70；相机位置与视点使用相同 `z`，禁止水平侧角，以更强但仍保留第三人称深度的纵向俯角观察，并以 35% 比例轻跟主角横移。
- 下车门位于末段右侧壁，屏幕上方偏右恒为出口；正前方端墙必须明确封闭，主角稳定占据下方中央约 22–30% 高度，车厢不建天花板。
- 前景：固定“你 / YOU”标签、浅色头部、红色背包与随人物移动的脚底光环；中景：可挤压的人群和扶手；背景：右侧黄色出口横梁、封闭端墙、窗、广告与路线灯。
- 顶部 HUD 避让安全区 10 px；临时摇杆出现在任意落指位置，松手消失，不永久占据左下角。
- 注意力路径：主角暖色局部光与脚底光环 → “你 / YOU”标签 → 右上“出口”转向箭标 → 地面折线箭头与人群缝隙 → 倒计时与摇晃预警格。

## 3. Color

- 墨黑紫 `#1D1022`：外框、轮廓、最暗阴影。
- 深茄紫 `#3E1747`：车体暗面、HUD、外部隧道。
- 旧纸奶油 `#F1E5C5`：墙面高光、主文字、玩家识别色。
- 酸性黄 `#F3F02D`：出口、方向箭头、主要按钮、稳定状态。
- 印刷朱红 `#ED4A37`：关门、危险、摔倒与失败。
- 柜台绿转译为车厢绿 `#2E5B45`：座椅、地板分区和环境锚点。
- 冷柜青转译为车窗青 `#65B9AE`：窗外反光和少量金属边缘。
- UI 比例：墨黑/深紫 42%，奶油 26%，车厢绿 17%，青 7%，酸黄 5%，朱红 3%。3D 场景整体降低饱和度与曝光，酸黄不在环境里大面积铺开；禁止紫蓝渐变与大面积荧光。

## 4. Typography

- 标题与关卡章：Impact、Arial Black、`Noto Sans SC`/`PingFang SC` 粗体回退，倾斜 -3°，2–4 px 硬墨影。
- UI/正文：Onest 或系统黑体 600/800；数字：JetBrains Mono / ui-monospace 700。
- 标题 40–56 px，屏幕标题 28–36 px，正文 16–18 px，HUD 11–15 px，按钮 18–22 px。
- 英文可全大写并增加 `0.06em` 字距；中文不强制加字距。

## 5. Shape, material, and lighting

- 车厢保持准确的长矩形比例，但墙板、滑动门扇、玻璃、橡胶地板、金属立柱和塑料座椅使用 `MeshStandardMaterial`；只有 DOM 使用 3–5 px 粗黑边、0–5 px 小圆角和 4–7 px 硬阴影。
- 车厢环境关闭几何描边壳，金属使用约 `roughness 0.34 / metalness 0.72`，塑料与橡胶约 `roughness 0.72 / metalness 0.04`；不增加漫画网点、速度线或整场描边滤镜。
- CSS 网点只覆盖 HUD 纸面和极少量画面边缘，不能进入 3D 材质或压住人物。
- Canvas 使用 ACES Filmic、曝光 0.96；基础半球光 0.64、暖主光 1.46、冷填光 0.54。车厢可比上一版稍暗，但乘客服装、座椅、立柱和通路必须在窄屏截图中清楚可见，禁止大片死黑。
- 5 组暖白灯安装在车体节点内，跟随车厢滚转；亮度以 0.72 Hz 缓慢呼吸，一盏松动灯以 2.15 Hz 轻微不稳，晃动峰值部分灯短暂跌至 38%。禁止高频闪烁和全屏白闪。
- 主角自带强度 2.35、半径 4.2 的暖色 PointLight，加法混合光环只围绕脚底并跟随人物；它必须在地面和邻近乘客上产生可见照明变化，而不是屏幕空间贴纸。
- 侧门出口使用强度 2.9、半径 5.3 的暖色 PointLight 与强度 1.45、半径 3.5 的绿色 PointLight；三盏绿色门状态灯、黄色门框和地面箭头组成从远处到门槛的连续引导。

## 6. Characters, environments, and assets

- 人物复用现有低多边形角色比例：头:躯干:腿约 `1:1.3:1.5`，方块身形、低段圆柱关节与夸张职业特征；主角整体缩放为 0.90，普通/宽体角色基础缩放为 0.84，小体型与幽灵为 0.82，以便俯视镜头显示更多通道。
- 玩家为浅色头部、易读外套、深紫裤、朱红背包、双层脚底光环、真实局部光与屏幕空间“你 / YOU”标签；人物身体不添加黑色外描边。识别主因是空间照明、背包、光环和标签，不要求整座场景迁就一套夸张漫画配色。乘客不得同时拥有这组完整识别组合。
- 乘客需有 4 种肤色、3 种体型和至少 8 套服装；每关改变主乘客池与密度。
- 后期关卡把现有怪兽 roster 转译成通勤乘客：吸血鬼、僵尸、木乃伊、骷髅、幽灵与狼人保留一眼可认的头部轮廓和职业道具；人物仍可使用 toon 色阶，但必须接受车厢灯和主角局部光，拥有一致接触阴影。
- 车厢必须包含：两排座椅、双片滑动门扇与门槽、绿色门状态灯、深色门玻璃、金属窗框、路线图、广告、6 根立柱、8 个吊环、橡胶地板接缝、防滑条、门槛警示纹与车厢编号。
- 人物模型与环境模型均为运行时几何；人物保留低多边形表面，环境改用写实 PBR 材质与更密集的结构接缝。不混入照片人物，不给场景叠漫画网点或全局描边。

## 7. UI and icons

- 图标统一为自绘 24×24 SVG、2 px 墨线、方角/圆端混合：暂停、声音、继续、重试和方向。
- HUD 是顶部三联票据：关卡、剩余秒数、离门距离。奶油底、粗墨边、无玻璃拟态。
- 排行榜使用同一张印刷票据语言：数字名次章、圆形头像、姓名和等宽分数；其他玩家的头像姓名为可点击组合，自身行仅显示强调色“你”。
- 主按钮酸黄底、3 px 墨边、5 px 硬影；按下下移 3 px并缩短阴影，80 ms。
- 首屏直接显示真实游戏场景，不出现封面菜单或开始按钮；引导复用《Block Party》的动态图式：120 px 虚线圆环、44 px 发光虚拟拇指、2.2 秒四向绕圈和“任意位置拖动移动”，首次真实输入即消失。
- 实际摇杆同样复用《Block Party》的轻量半透明图式：120 px 白色透明圆环、56 px 白色拇指、无十字线与硬漫画投影；它是用户明确指定的控制层例外，HUD 和结果界面继续保持漫画风格。
- 危险态同时使用朱红、倾斜边框、警告文字和脉冲，不只靠颜色。
- 禁用态使用斜线网点与明确标签；聚焦态为 3 px 奶油/酸黄双框。
- Emoji 政策：功能图标、HUD 和教程均不使用 Emoji。

## 8. Motion and VFX

- 所有站立人物都带有《Block Hop》式“有弹性的呼吸备战”：约 0.98 秒一次轻跳呼吸，主角振幅 0.075、乘客 0.045 world-unit；乘客相位错开，不能整车同步上下。手臂随呼吸微抬、腿部轻微换重心，移动时叠加 7 Hz 步伐弹跳。
- 人物行走为 5.5–7 Hz 手脚反摆；挤压接触 70 ms，恢复 160 ms。
- 晃动预警 620–1150 ms：吊环先摆、车体先倾 35%，DOM 警告格两次错版位移；冲击峰值 90 ms，车体回弹 420 ms。
- 普通侧摔约 0.82–1.02 秒；翻跟斗约 1.08–1.58 秒，包含最高 0.48 world-unit 离地弧线、完整 360° 前滚和 0.18 rad 侧摆。玩家倒地期间继续计时，但不使用长时间镜头震动。
- 碰撞纸屑 4–8 枚、寿命 280–460 ms；过关 12–18 枚酸黄/奶油纸屑，寿命 520–760 ms。
- 减弱动态：取消 DOM 错位、相机冲击和纸屑，仅保留车体 40% 倾斜、人物静态倒地与文字状态。

### 反馈矩阵

| 事件 | 即时确认 | 结果与反馈 | 强度 | 恢复 |
|---|---|---|---:|---|
| 移动 | 玩家从原地呼吸弹跳过渡到 7 Hz 步伐弹跳、脚步摆动 | 速度平滑提升，轻脚步音 | 1 | 松手 180 ms 回到呼吸备战 |
| 推挤乘客 | 双方压缩、短墨线爆点 | 法向分离并交换动量 | 2 | 160–300 ms 回正 |
| 站稳 | 脚下酸黄半环闭合 | 晃动失衡降低 32% | 2 | 移动即取消 |
| 晃动预警 | 吊环同向摆、朱红提示格 | 玩家有 0.62–1.15 s 应对 | 3 | 冲击后提示退出 |
| 摔倒 | 90 ms 冲击位移、朱红星爆 | 失去控制 0.8–1.4 s | 4 | 起身后 260 ms 保护 |
| 抵达车门 | 门区酸黄反转、纸屑 | 计时冻结并结算 | 4 | 下一关按钮 |

## 9. References translated into principles

- 《午夜验客》：继承粗墨、旧纸、酸黄/朱红双强调与网点印刷；不复制便利店柜台和怪谈人物插画。
- 《Get Off My Grave》：继承斜俯视 3D 群体、明确接触阴影和角色 roster；不复制墓园雾、塔防 HUD 或暗色发光材质。
- 早高峰地铁：借用无顶棚剖面、纵向通道和吊环同步摆动；不追求线路品牌或城市写实还原。

## 10. Anti-patterns

- 禁止默认圆角玻璃卡、霓虹渐变、柔光按钮、Emoji 和混搭图标。
- 禁止只有空盒子和人物方块而缺少门、窗、扶手、吊环、座椅、广告和线路信息。
- 禁止车厢继续使用奶油色玩具塑料、环境粗黑描边或整块无接缝墙板；写实感必须来自结构比例、材质响应、门窗五金和地板接缝。
- 禁止镜头跟着每次碰撞剧烈抖动；禁止晃动无预警随机惩罚。
- 禁止乘客服装高饱和色平均铺满，导致主角局部光、背包和出口无法识别。
- 禁止把漫画边框、网点、拟声字或平面发光滤镜烘进 3D 地铁；禁止用全场高曝光代替局部灯光层次。
- 禁止在主角身体、头发、四肢或背包外再包一层黑色放大网格作为描边；脚底识别环的深色外环不属于人物描边。
- 禁止 UI 像独立 App 浮在 3D 世界上；所有面板必须像同一本漫画的票据与格框。

## 11. Vertical-slice acceptance

- 开场：页面加载即看到第三人称真实车厢、下方中央主角、上方偏右侧门与幽灵手指，不出现开始菜单。
- 游戏：390×844 首关中玩家、右上侧门、折线路径、乘客、动态摇杆、倒计时和晃动预警同时可读。
- 高反馈：一次预警→冲击→多人互撞→至少一人摔倒完整可见，玩家能理解原因。
- 完成：过关提供“下一关”，错过站提供“再挤一次”与排行榜；不存在最终通关画面。
- 后期：怪兽乘客在拥挤人群中可辨识，但不抢走玩家和酸黄出口的第一视觉层级。
- 窄屏：320×568 主角仍是第一焦点，出口提示位于 HUD 下方，任意位置摇杆不被固定裁切。
- 视觉 QA：首轮截图后修复所有 P0/P1，并以相同状态和尺寸复验。
