# 《挤下地铁》视觉圣经

> 已锁定方向：界面参考现有《Block Hop / 跳一跳》的轻量游戏 UI；人物保留现有低多边形 roster，3D 地铁车厢采用写实 PBR 材质与轨道交通结构，用昏暗摇晃的车厢灯、出口引导光和主角局部光建立空间。UI 使用奶油胶囊、圆润字形、深色细边与黄色主操作，不覆盖场景。

## 1. Visual thesis

- 游戏与受众：移动端短局动作玩家，熟悉早高峰但希望获得夸张喜剧释放。
- 情绪承诺：昏暗末班车里的真实压迫感，被一个清楚、圆润的大倒计时切成紧张但轻快的节拍。
- 一句话视觉命题：一节被摇晃顶灯切出明暗层次的无顶棚地铁里，带红色通勤识别件与随身暖光的主角挤向门口，像《跳一跳》一样克制的胶囊 UI 只负责告诉玩家时间、距离与危险。
- 标志性视觉时刻：车身先倾、吊环齐摆，朱红圆角警告条快速弹入，全车乘客像多米诺骨牌般互撞摔倒。
- 三个必须品质：车厢细节可信、动作一眼可读、混乱仍有视觉层级。
- 三个避免方向：默认 Three.js 灰盒、日式二次元、霓虹赛博地铁。

## 2. Composition and camera

- 竖屏 320×568–430×932；3D Canvas 全屏，DOM HUD 自适配，不整页缩放。
- 55° 透视相机位于主角后方 5.2 world-unit、高 6.05 world-unit，严格沿车厢 `+X` 中轴看向前方 4.8 world-unit、视点高度 0.70；相机位置与视点使用相同 `z`，禁止水平侧角，以更强但仍保留第三人称深度的纵向俯角观察，并以 35% 比例轻跟主角横移。
- 车厢末段左右侧壁各有一套位置、尺寸和结构完全一致的候选车门与窗组；每关只有一侧开门，首关固定右侧、此后左右交替。正前方端墙必须明确封闭，主角稳定占据下方中央约 22–30% 高度，车厢不建天花板。
- 前景：主角红色通勤识别件与随人物移动的暖色局部光；中景：可挤压的人群和扶手；背景：左右对称门窗、封闭端墙、广告与路线灯。正确侧开门亮绿灯，错误侧闭门亮红灯；禁止脚底识别圈以及固定在屏幕中心、无法跟随世界坐标的“你 / YOU”头顶标记。
- 顶部大倒计时避让安全区 10 px；临时摇杆出现在任意落指位置，松手消失，不永久占据左下角。
- 注意力路径：主角暖色局部光与红色识别件 → 正确门前随车厢运动的绿色 3D 箭头 → 绿色开门与地面折线路径 → 倒计时与摇晃警告条。

## 3. Color

- 柔墨黑 `#15110E`：UI 边线、文字与小幅硬影。
- 暗紫褐 `#302633`：遮罩与车体外部暗面，不作为大面积 HUD 底色。
- 奶油白 `#FFF7E6`：胶囊 HUD、卡片与主要浅色表面。
- 暖纸黄 `#FFEEC4`：次级卡片和轻微层次。
- 跳跃黄 `#FFCE2E`：出口、方向箭头与主要按钮。
- 警告红 `#FF4438`：最后 5 秒、摔倒与失败标题。
- 跳跃青 `#3FB6AC`：过关、排行辅助与次级成功状态。
- 柜台绿转译为车厢绿 `#2E5B45`：座椅、地板分区和环境锚点。
- 冷柜青转译为车窗青 `#65B9AE`：窗外反光和少量金属边缘。
- UI 比例：奶油白 48%，柔墨黑 26%，跳跃黄 12%，跳跃青 8%，警告红 6%。3D 场景继续使用自身低饱和材质；禁止紫蓝渐变与大面积荧光。

## 4. Typography

- 数字与标题：`Arial Rounded MT Bold`、`Avenir Next`、`PingFang SC` 800–900 回退；不再使用 Impact、Arial Black 或多层文字描边。
- UI/正文：`Avenir Next`、`PingFang SC`、系统黑体 600–800；数字启用 `tabular-nums`。
- 倒计时 54–68 px，屏幕标题 38–48 px，正文 15–17 px，HUD 标签 10–12 px，按钮 18–22 px。
- 英文标签可全大写并增加 `0.08em` 字距；中文保持自然字距。文字只允许 0–2 px 的轻影，不使用 4 px 描边或夸张倾斜。

## 5. Shape, material, and lighting

- 车厢保持准确的长矩形比例，但墙板、滑动门扇、玻璃、橡胶地板、金属立柱和塑料座椅使用 `MeshStandardMaterial`；DOM 参考《跳一跳》使用 2.5–4 px 深色边、8–999 px 圆角和 2–6 px 小硬影。
- 车厢环境关闭几何描边壳，金属使用约 `roughness 0.34 / metalness 0.72`，塑料与橡胶约 `roughness 0.72 / metalness 0.04`；不增加漫画网点、速度线或整场描边滤镜。
- UI 不再叠全屏网点或漫画边框；3D 材质与人物保持原样。
- Canvas 使用 ACES Filmic、曝光 0.96；基础半球光 0.64、暖主光 1.46、冷填光 0.54。车厢可比上一版稍暗，但乘客服装、座椅、立柱和通路必须在窄屏截图中清楚可见，禁止大片死黑。
- 5 组暖白灯安装在车体节点内，跟随车厢滚转；亮度以 0.72 Hz 缓慢呼吸，一盏松动灯以 2.15 Hz 轻微不稳，晃动峰值部分灯短暂跌至 38%。禁止高频闪烁和全屏白闪。
- 主角自带强度 2.35、半径 4.2 的暖色 PointLight，跟随人物在地面和邻近乘客上产生可见照明变化；不再绘制 Torus、光晕或其他脚底识别圈，也不使用屏幕空间贴纸。
- 正确侧门使用强度 2.9、半径 5.3 的暖色 PointLight 与强度 1.45、半径 3.5 的绿色 PointLight，三盏绿灯、打开的门扇、地面折线路径和世界空间绿色 3D 箭头组成连续引导；错误侧门扇闭合，三盏红灯与低强度红光明确表达不可通行。箭头使用高明度薄荷绿主体与深绿发光材质，以 1.35 Hz 上下浮动 0.08 world-unit，并随车厢一起摇晃，不使用二维 HUD 出口标记。

## 6. Characters, environments, and assets

- 人物复用现有低多边形角色比例：头:躯干:腿约 `1:1.3:1.5`，方块身形、分段肘膝关节与夸张职业特征；主角、站立乘客、职业乘客、怪物乘客和坐席乘客的外层缩放统一为 0.58，儿童与小体型只通过内部头身比和腿长产生自然身高差，不允许再把整个人缩成远小于主角的棋子。
- 可选主角首版为 10 名：通勤者、警察、护士、消防员、厨师、快递员、僵尸、吸血鬼、猫和狗。人物/怪物继续使用共享分段低多边形骨架，猫狗使用共享方块动物比例；每名主角都有朱红背包、肩带、围巾或项圈之一作为通勤识别件，并共享真实局部光。人物身体不添加黑色外描边，不显示脚底圈或屏幕空间头顶标签。乘客不得同时拥有“红色识别件 + 主角暖光”的完整组合。
- 乘客直接使用共享人物库的市民与职业身份，不把颜色排列当成新造型：上班族、办公室女性、学生、老人、店员、厨师、警察、护士、消防员、建筑工、外卖员、主管、快递员、清洁工、咖啡师和保安必须通过不同帽型、制服、围裙、反光带、背包、徽章、工具或手提箱形成可读轮廓。所有座椅只允许随机出现 1–2 个空位，其余座位必须有人；坐席按阅读、看手机和休息三种活动分布。
- 站立乘客姿态必须构成真实车厢切片：抓吊环者一侧肩臂完整抬高并对准加长后的吊带，阅读者双臂托书、胸口和视线轻低，通话者单手贴近耳侧，看手机者双臂在胸前收拢；只有 2–4 名随机换位者使用完整步态。首关只保留 9 名站立乘客，让这些姿态和职业轮廓之间有足够负空间。
- 后期关卡把现有怪兽 roster 转译成通勤乘客：吸血鬼、僵尸、木乃伊、骷髅、幽灵与狼人保留一眼可认的头部轮廓和职业道具；人物仍可使用 toon 色阶，但必须接受车厢灯和主角局部光，拥有一致接触阴影。
- 车厢必须包含：两排座椅、左右对称的双片滑动门扇与门槽、红绿门状态灯、深色门玻璃、金属窗框、路线图、广告、6 根立柱、8 个吊环、橡胶地板接缝、防滑条、门槛警示纹与车厢编号。
- 人物模型与环境模型均为运行时几何；人物保留低多边形表面，环境改用写实 PBR 材质与更密集的结构接缝。不混入照片人物，不给场景叠漫画网点或全局描边。

## 7. UI and icons

- 图标统一为自绘 24×24 SVG、2 px 柔墨线和圆端：暂停、继续、重试、关闭与方向。
- 游戏中信息层只常驻顶部居中的奶油白胶囊：54–68 px 整数秒倒计时、紧随其下的实时距门距离；右上暂停为 46–48 px 圆形按钮。不常驻关卡、摔倒数或分数。最后 5 秒数字改为警告红并通过缩放跳动形成非颜色提示。
- 关卡编号与车厢名只在进入新关时以小型奶油胶囊出现 2.4 秒，随后淡出，不占用持续注意力。
- 结算保留现有信息结构，但改为《跳一跳》式奶油圆角卡：成功/失败标题、两列统计、黄色胶囊主按钮；移除卡片倾斜、网点底纹和粗重标题描边。
- 角色收藏使用同一套奶油圆角外壳，但主体改成《过马路》式 3D 角色站台：画面约 62% 高度用于直接展示真实模型，中央当前角色完整、最大、最亮，两侧各露出一名缩小角色作为可滑动线索；角色脚下使用扁平奶油展示台而非游戏中的识别圈。左右圆形箭头、`序号/总数` 和底部短横进度共同表达可循环浏览。信息区只保留当前角色名称、类别与拥有/价格状态；唯一大按钮在“解锁 / 装备 / 使用中 / 金币不足”之间切换，不再陈列两列文字卡。
- 排行榜使用相同的奶油圆角卡、圆形头像、姓名与清晰数字；其他玩家的头像姓名为可点击组合，自身行仅显示强调色“你”。
- 主按钮跳跃黄底、3 px 柔墨边、4 px 小硬影、999 px 圆角；按下下移 2 px并缩短阴影，80 ms。
- 首屏直接显示真实游戏场景，不出现封面菜单或开始按钮；引导复用《Block Party》的动态图式：120 px 虚线圆环、44 px 发光虚拟拇指、2.2 秒四向绕圈和“任意位置拖动移动”，首次真实输入即消失。
- 实际摇杆同样复用《Block Party》的轻量半透明图式：120 px 白色透明圆环、56 px 白色拇指、无十字线与硬漫画投影；它是用户明确指定的控制层例外，HUD 和结果界面继续保持《跳一跳》的奶油胶囊体系。
- 危险态同时使用警告红、警告文字和两段缩放，不只靠颜色。
- 禁用态使用降低对比度与明确标签；聚焦态为 3 px 奶油/黄色外框。
- Emoji 政策：功能图标、HUD 和教程均不使用 Emoji。

## 8. Motion and VFX

- 所有静止人物都带有《Block Hop》式“有弹性的呼吸备战”：约 0.98 秒一次轻跳呼吸，主角振幅 0.052、乘客 0.032 world-unit；乘客相位错开，不能整车同步上下。手臂随呼吸微抬、腿部轻微换重心。
- 坐席乘客不使用站立弹跳：髋部固定在座面，躯干竖直，大腿与躯干形成清楚的 90° 坐姿并向通道伸出，小腿从膝盖垂下，鞋尖继续朝通道；任何观察角度都不能让腿脚切进坐垫。胸口以 1.6–2.4 秒周期起伏不超过 0.012 world-unit，头肩随车体横摆不超过 1.8°；阅读者保留摊开的双色封面与纸页，看手机者低头，休息者自然呼吸。伸出的膝脚同时构成安静但真实的通道边缘障碍，站立乘客默认不换位，只有 2–4 名随机换位者具有与实际路程绑定的步态。
- 被选为下车者的坐席使用“察觉—准备—起身—跨入通道—快走”的连续动作：0.72 秒内收起书/手机、抬头、上身前倾并收脚，0.58 秒内从髋部发力、伸膝站起并向通道跨一步；进入通道后立刻使用与实际路程绑定的正常步态朝门移动。动作不使用高亮头顶符号、屏幕警告或瞬移，玩家依靠身体前倾、物品收起和座椅摩擦声预判。
- 行走动画与实际地面路程绑定，不再使用固定 Hz：主角每 1.05 world-unit、乘客每 0.58 world-unit 完成一轮左右脚；腿从髋部枢轴摆动，身体起伏、手臂反摆和转身使用同一相位。停下 120 ms 内收腿回正；挤压接触 70 ms，恢复 160 ms。
- 晃动预警 620–1150 ms：吊环先摆、车体先倾 35%，DOM 警告条进行 0.985–1.02 的两段缩放；冲击峰值 90 ms，车体回弹 420 ms。
- 普通侧摔约 0.94–1.10 秒；前扑摔倒约 1.00–1.34 秒，使用重心前移、补偿步、逐级屈膝、近侧手先伸、另一手延迟补撑、肩胯缓冲、短接地、单膝支撑、推地抬髋和两段伸膝共 11 个姿态节点。前 46% 时间快速失衡落地，9% 接触缓冲，后 45% 清晰起身；上臂—前臂—手掌、大腿—小腿和头颈均有独立枢轴。节点间使用连续速度的三次插值，根姿态倾角限制在约 20–22°，落地主要由胸背、髋、膝、肘和前臂完成。主角局部灯保持世界空间稳定，不随身体翻转。猫狗以四足压低、侧翻、前爪撑地和重新站稳替代人形屈膝动作。禁止慢动作悬停、完整根节点翻转、逐节点停顿、僵硬直腿落地、整体线性弹起和任何穿地帧。
- 人形角色外层视觉尺寸统一为 0.58；职业轮廓允许因帽子、背包和体型在高度或宽度上产生差异，但普通乘客不得系统性小于人形主角。猫狗允许更低、更长的自然四足轮廓，但碰撞半径仍为 0.22；主角依靠红色通勤识别件与暖光而不是体型优势识别。
- 碰撞纸屑 4–8 枚、寿命 280–460 ms；过关 12–18 枚酸黄/奶油纸屑，寿命 520–760 ms。
- 减弱动态：取消 DOM 缩放、相机冲击和纸屑，仅保留车体 40% 倾斜、人物静态倒地与文字状态。

## 9. References translated into principles

- 《Block Hop / 跳一跳》：继承奶油胶囊、柔墨细边、小硬影与黄色主操作；不复制其低多边形户外场景或粗重展示字体。
- 《Get Off My Grave》：继承斜俯视 3D 群体、明确接触阴影和角色 roster；不复制墓园雾、塔防 HUD 或暗色发光材质。
- 早高峰地铁：借用无顶棚剖面、纵向通道和吊环同步摆动；不追求线路品牌或城市写实还原。

## 10. Anti-patterns

- 禁止默认圆角玻璃卡、霓虹渐变、柔光按钮、Emoji 和混搭图标。
- 禁止只有空盒子和人物方块而缺少门、窗、扶手、吊环、座椅、广告和线路信息。
- 禁止车厢继续使用奶油色玩具塑料、环境粗黑描边或整块无接缝墙板；写实感必须来自结构比例、材质响应、门窗五金和地板接缝。
- 禁止镜头跟着每次碰撞剧烈抖动；禁止晃动无预警随机惩罚。
- 禁止乘客服装高饱和色平均铺满，导致主角局部光、背包和出口无法识别。
- 禁止整排座椅空置；禁止大多数站立乘客在出生点周围随机游走、反复转身或没有目的地横穿通道；禁止整车人物共用自然垂臂站姿，也禁止只换衣服颜色却缺少职业轮廓和生活行为。
- 禁止把漫画边框、网点、拟声字或平面发光滤镜烘进 3D 地铁；禁止用全场高曝光代替局部灯光层次。
- 禁止在主角身体、头发、四肢或背包外再包一层黑色放大网格作为描边；主角脚底不允许出现深色环、发光环或半透明光晕。
- 禁止 UI 像独立 App 浮在 3D 世界上；所有面板必须属于同一套奶油胶囊与圆角卡系统。
